Block si Inline

Introducere

In lectia anterioara, am descoperit ca diferitele tipuri de afisare au modele de casete distincte si putem modifica calculul casetei prin schimbarea proprietatii box-sizing. CSS are doua tipuri de casete: block si inline, care determina comportamentul si interactiunea elementelor. Proprietatea display controleaza modul in care elementele HTML apar pe pagina web. Vom explora mai detaliat optiunile sale in aceasta lectie.

Block vs inline

Majoritatea elementelor pe care le-ai invatat pana acum sunt elemente de tip block. Cu alte cuvinte, stilul lor implicit este display: block. In mod implicit, elementele block vor aparea pe pagina stivuite unele peste altele, fiecare element nou incepand pe o linie noua.

Elementele inline, pe de alta parte, nu incep pe o noua linie. Ele apar in rand cu elementele langa care sunt plasate. Un exemplu clar de element inline este un link, adica tag-ul <a>. Daca inserezi un astfel de tag in mijlocul unui paragraf de text, link-ul se va comporta ca parte din acel paragraf. In plus, padding si margin se comporta diferit pe elementele inline. In general, nu este recomandat sa adaugi padding sau margin suplimentar elementelor inline.

Compromisul: inline-block

Elementele inline-block se comporta ca elementele inline, dar permit utilizarea padding si margin in stil block. display: inline-block este un instrument util de cunoscut, dar in practica, probabil vei folosi mai des flexbox daca vrei sa aliniezi mai multe casete. Flexbox va fi explicat in detaliu in lectia urmatoare.

Div-uri si span-uri

Nu putem vorbi despre elementele block si inline fara sa discutam despre div-uri si span-uri. Toate celelalte elemente HTML pe care le-am intalnit pana acum ofera un sens continutului lor. De exemplu, elementele <p> ii spun browserului sa afiseze textul pe care il contin ca un paragraf. Elementele <strong> indica browserului ce texte sunt importante si asa mai departe. Totusi, div-urile si span-urile nu atribuie niciun sens specific continutului lor. Sunt doar casete generice care pot contine orice.

Disponibilitatea unor astfel de elemente este mult mai utila decat pare la prima vedere. Deseori vom avea nevoie de elemente care sa nu aiba alt scop decat sa fie „hook”-uri, astfel incat sa le putem atribui un id sau o clasa pentru a le stiliza cu CSS. Un alt caz de utilizare frecvent este gruparea unor elemente conexe sub un element parinte, pentru a le pozitiona corect pe pagina. Div-urile si span-urile ne permit sa facem acest lucru.

<div> este un element de tip block in mod implicit. Este folosit frecvent ca element container pentru gruparea altor elemente. Div-urile ne permit sa impartim pagina in diverse blocuri si sa aplicam stilizari acestor blocuri.

Span este un element de tip inline in mod implicit. Poate fi folosit pentru a grupa continutul textului si elementele HTML inline pentru stilizare si ar trebui utilizat doar atunci cand niciun alt element HTML semantic nu este potrivit.
