Introducere in CSS

Introducere

In lectia anterioara, ai invatat cum sa scrii HTML-ul care determina structura unei pagini web. Urmatorul pas este sa faci ca aceasta structura sa arate bine, adaugand stil, iar exact pentru acest lucru este destinat CSS. In urmatoarele lectii, ne vom concentra pe concepte fundamentale ale CSS, lucruri pe care toata lumea ar trebui sa le cunoasca de la inceput—fie ca abia incep sau au nevoie de o recapitulare.

Sintaxa de baza

La cel mai elementar nivel, CSS este format din diverse reguli. Aceste reguli sunt alcatuite dintr-un selector (vom discuta mai multe despre acesta mai tarziu) si o lista de declaratii separate prin punct si virgula, fiecare dintre acestea fiind compusa dintr-o pereche proprietate-valoare.

Selectori

Selectorii se refera la elementele HTML carora li se aplica regulile CSS; ei sunt ceea ce este efectiv „selectat” pentru fiecare regula. Subsectiunile urmatoare nu acopera toti selectorii disponibili, dar sunt de departe cei mai comuni si cei cu care ar trebui sa te familiarizezi mai intai.

Selector universal

Selectorul universal va selecta elemente de orice tip (adica din intregul document), de unde si numele „universal”, iar sintaxa pentru acesta este un simplu asterisc. In exemplul de mai jos, fiecare element ar avea stilul color: purple; aplicat. 

                
                    * {
                        color: purple;
                    }

Selectori de tip

Un selector de tip (sau selector de element) va selecta toate elementele de un anumit tip, iar sintaxa este pur si simplu numele elementului: 

                
                    <!-- index.html -->

                    <div>Hello, World!</div>
                    <div>Hello again!</div>
                    <p>Hi...</p>
                    <div>Okay, bye.</div>

                
            
                
                    /* styles.css */

                    div {
                        color: white;
                    }

Aici, toate cele trei <div> elemente ar fi selectate, in timp ce elementul <p> nu ar fi.

Selectori de clasa

Selectorii de clasa vor selecta toate elementele care au o anumita clasa, care este doar un atribut adaugat unui element HTML. Iata cum poti adauga o clasa unui tag HTML si sa o selectezi in CSS: 

                
                    <!-- index.html -->

                    <div class="alert-text">Please agree to our terms of service.</div>

                
            
                
                    /* styles.css */

                    .alert-text {
                        color: red;
                    }

Observa sintaxa selectorilor de clasa: un punct urmat imediat de valoarea sensibila la majuscule a atributului class. Clasele nu trebuie sa fie specifice unui anumit element, astfel incat poti folosi aceeasi clasa pe cate elemente doresti.

Un alt lucru pe care il poti face cu atributul class este sa adaugi mai multe clase unui singur element, sub forma unei liste separate prin spatii, de exemplu: class="alert-text severe-alert". Deoarece spatiul este folosit pentru a separa numele claselor in acest mod, nu ar trebui sa utilizezi spatii pentru denumiri compuse din mai multe cuvinte; in schimb, foloseste o liniuta (-).

Selectori ID

Selectori ID Selectorii ID sunt similari cu selectorii de clasa. Ei selecteaza un element cu un anumit ID, care este un alt atribut pe care il adaugi unui element HTML. Principala diferenta intre clase si ID-uri este ca un element poate avea un singur ID. Acesta nu poate fi repetat pe o singura pagina si nu ar trebui sa contina spatii: 

                
                    <!-- index.html -->

                    <div id="title">My Awesome 90's Page</div>

                
            
                
                    /* styles.css */

                    #title {
                        background-color: red;
                    }

Selectorul de grupare

Ce se intampla daca avem doua grupuri de elemente care impartasesc unele dintre declaratiile lor de stil? 

                
                        .read {
                            color: white;
                            background-color: black;
                            /* alte declaratii */
                        }

                        .unread {
                            color: white;
                            background-color: black;
                            /* alte declaratii */
                        }

Atat selectorii .read cat si .unread impartasesc declaratiile color: white; si background-color: black;, dar altfel au mai multe declaratii unice. Pentru a reduce repetitia, putem grupa acesti doi selectori impreuna intr-o lista separata prin virgula: 

                
                    .read,
                    .unread {
                        color: white;
                        background-color: black;
                    }

                    .read {
                        /* alte declaratii */
                    }

                    .unread {
                        /* alte declaratii */
                    }

Ambele exemple de mai sus (cu si fara grupare) vor avea acelasi rezultat, dar al doilea exemplu reduce repetitia declaratiilor si face mai usor de modificat fie color, fie background-color pentru ambele clase simultan.

Inlantuirea selectorilor

O alta modalitate de a folosi selectori este sa Ii Inlantuiesti Intr-o lista fara nicio separare. Sa presupunem ca avem urmatorul cod HTML: 

                
                    <div>
                        <div class="subsection header">Latest Posts</div>
                        <p class="subsection preview">This is where a preview for a post might go.</p>
                    </div>

Avem doua elemente cu clasa subsection, care au unele stiluri unice, dar daca vrem sa aplicam o regula separata doar elementului care are si clasa header ca a doua clasa? Ei bine, putem Inlantui ambii selectori de clasa In CSS astfel: 

                
                    .subsection.header {
                        color: red;
                    }

Ceea ce face .subsection.header este ca selecteaza orice element care are atat clasa subsection, cat si clasa header. Observa ca nu exista spatiu Intre selectorii .subsection si .header. Aceasta sintaxa functioneaza practic pentru Inlantuirea oricarei combinatii de selectori, cu exceptia cazului In care sunt mai multi selectori de tip.

Aceasta poate fi folosita si pentru a Inlantui o clasa si un ID, asa cum se arata mai jos: 

                
                    <div>
                        <div class="subsection header">Latest Posts</div>
                        <p class="subsection" id="preview">
                            This is where a preview for a post might go.
                        </p>
                    </div>

Poti lua cele doua elemente de mai sus si le poti combina astfel: 

                
                    .subsection.header {
                        color: red;
                    }

                    .subsection#preview {
                        color: blue;
                    }

In general, nu poti inlantui mai multi selectori de tip, deoarece un element nu poate fi simultan doua tipuri diferite. De exemplu, inlantuirea a doi selectori de tip precum div si p ar produce selectorul divp, care nu ar functiona, deoarece ar incerca sa gaseasca un element , care nu exista.

Descendant combinator

Combinatorii ne permit sa combinam mai multi selectori in moduri diferite fata de grupare sau inlantuire, deoarece arata o relatie intre selectori. Exista patru tipuri de combinatori in total, dar pentru moment ne vom concentra doar pe descendant combinator, care este reprezentat in CSS printr-un spatiu intre selectori. Un descendant combinator va selecta elementele care corespund ultimului selector doar daca acestea au un stramos (parinte, bunic etc.) care corespunde selectorului anterior.

De exemplu, .ancestor .child ar selecta un element cu clasa child daca acesta are un stramos cu clasa ancestor. O alta modalitate de a gandi acest lucru este ca child va fi selectat doar daca este imbricat in ancestor, indiferent de cat de adanca este aceasta imbricare. Arunca o privire rapida la exemplul de mai jos si vezi daca poti identifica care elemente ar fi selectate pe baza regulii CSS furnizate. 

                
                    <!-- index.html -->

                    <div class="ancestor">
                        <!-- A -->
                        <div class="contents">
                            <!-- B -->
                            <div class="contents"><!-- C --></div>
                        </div>
                    </div>

                    <div class="contents"><!-- D --></div>

                
            
                
                    /* styles.css */

                    .ancestor .contents {
                        /* alte declaratii */
                    }

In exemplul de mai sus, primele doua elemente cu clasa contents (B si C) ar fi selectate, dar ultimul element (D) nu ar fi. Ai ghicit?

Nu exista cu adevarat o limita pentru numarul de combinatori pe care ii poti adauga unei reguli, asa ca .one .two .three .four ar fi complet valid. Aceasta ar selecta un element care are clasa four daca are un stramos cu clasa three, iar acel stramos are propriul sau stramos cu clasa two, si asa mai departe. Totusi, in general, ar trebui sa eviti incercarea de a selecta elemente care necesita acest nivel de imbricare, deoarece poate deveni destul de confuz si lung, si poate cauza probleme legate de specificitate.

Proprietati pentru inceput

Exista cateva proprietati CSS pe care le vei folosi tot timpul sau, cel putin, destul de des. Vom introduce mai multe dintre aceste proprietati, desi aceasta nu este o lista completa. Invatarea urmatoarelor proprietati va fi suficienta pentru a incepe.

Color si background-color

Proprietatea color seteaza culoarea textului unui element, in timp ce background-color seteaza, ei bine, culoarea de fundal a unui element. Cred ca am terminat aici?

Aproape. Ambele proprietati pot accepta mai multe tipuri de valori. Una dintre cele mai comune este un cuvant cheie, precum un nume real de culoare, cum ar fi red, sau cuvantul cheie transparent. Acestea accepta si valori HEX, RGB si HSL, pe care poate le-ai intalnit daca ai folosit vreodata un program de editare foto sau un site unde iti poti personaliza culorile profilului. 

                
                    p {
                        /* hex exemplu: */
                        color: #1100ff;
                    }

                    p {
                        /* rgb exemplu: */
                        color: rgb(100, 0, 127);
                    }

                    p {
                        /* hsl exemplu: */
                        color: hsl(15, 82%, 56%);
                    }

Bazele tipografiei si text-align

font-family poate fi o valoare unica sau o lista separata prin virgula, care determina ce font foloseste un element. Fiecare font va apartine uneia dintre doua categorii: fie un „nume de familie de fonturi”, precum "Times New Roman" (folosim ghilimele din cauza spatiului dintre cuvinte), fie un „nume de familie generic”, precum serif (numele de familie generic nu utilizeaza ghilimele).

Daca un browser nu poate gasi sau nu accepta primul font din lista, va folosi urmatorul, apoi urmatorul si asa mai departe, pana cand gaseste un font acceptat si valid. De aceea, este o buna practica sa incluzi o lista de valori pentru aceasta proprietate, incepand cu fontul pe care doresti sa fie utilizat cel mai mult si terminand cu un font generic ca rezerva, de exemplu: font-family: "Times New Roman", serif;.

font-size va seta dimensiunea fontului, asa cum sugereaza numele proprietatii. Atunci cand atribui o valoare acestei proprietati, valoarea nu trebuie sa contina spatii, de exemplu: font-size: 22px nu are spatiu intre „22” si „px”.

font-weight afecteaza grosimea textului, presupunand ca fontul accepta greutatea specificata. Aceasta valoare poate fi un cuvant cheie, de exemplu: font-weight: bold, sau un numar intre 1 si 1000, de exemplu: font-weight: 700 (echivalent cu bold). De obicei, valorile numerice vor fi in trepte de 100 pana la 900, desi acest lucru va depinde de font.

text-align va alinia textul orizontal in cadrul unui element, iar poti folosi cuvintele cheie comune pe care le-ai intalnit in editoare de text ca valori pentru aceasta proprietate, de exemplu: text-align: center.

Inaltime si latime imagine

Imaginile nu sunt singurele elemente pentru care putem ajusta inaltimea si latimea, dar vrem sa ne concentram in mod specific asupra lor in acest caz.

Implicit, valorile height si width ale unui element <img> vor fi aceleasi ca dimensiunile reale ale fisierului imagine. Daca vrei sa ajustezi dimensiunea imaginii fara a-i pierde proportiile, trebuie sa folosesti valoarea auto pentru proprietatea height si sa ajustezi valoarea width: 

                
                    img {
                        height: auto;
                        width: 500px;
                    }

De exemplu, daca dimensiunea originala a unei imagini era 500px inaltime si 1000px latime, utilizarea CSS de mai sus ar rezulta intr-o inaltime de 250px.

Aceste proprietati functioneaza in conjunctie cu atributele height si width din HTML. Este recomandat sa incluzi atat aceste proprietati, cat si atributele HTML pentru elementele de imagine, chiar daca nu intentionezi sa ajustezi valorile fata de cele originale ale fisierului imagine. Daca aceste valori nu sunt incluse si o imagine se incarca mai lent decat restul continutului paginii, initial nu va ocupa spatiu pe pagina, dar va provoca brusc o schimbare drastica a continutului odata ce se incarca. Declararea explicita a unei inaltimi si latimi previne acest lucru, deoarece spatiul va fi „rezervat” pe pagina si va aparea gol pana cand imaginea se incarca.

Adaugarea CSS la HTML

Acum ca am invatat cateva elemente de sintaxa de baza, probabil te intrebi cum sa adaugam tot acest CSS in HTML-ul nostru. Exista trei metode prin care putem face acest lucru.

CSS extern

CSS extern este cea mai comuna metoda pe care o vei intalni si presupune crearea unui fisier separat pentru CSS si legarea acestuia in interiorul tag-urilor <head> de deschidere si inchidere ale HTML-ului, folosind un element <link> fara continut: 

                
                    <!-- index.html -->

                    <head>
                    <link rel="stylesheet" href="styles.css">
                    </head>

                
            
                
                    /* styles.css */

                    div {
                        color: white;
                        background-color: black;
                    }

                    p {
                        color: red;
                    }

Mai intai, adaugam un element <link> fara continut in interiorul tag-urilor de deschidere si inchidere <head> ale fisierului HTML. Atributul href reprezinta locatia fisierului CSS, fie un URL absolut, fie—ceea ce vei folosi—un URL relativ fata de locatia fisierului HTML. In exemplul nostru de mai sus, presupunem ca ambele fisiere sunt situate in acelasi director. Atributul rel este obligatoriu si specifica relatia dintre fisierul HTML si fisierul legat.

Apoi, in fisierul nou creat styles.css, avem selectorul (div si p), urmat de o pereche de acolade de deschidere si inchidere, care formeaza un „bloc de declaratii”. In final, plasam orice declaratii in interiorul blocului de declaratii. color: white; este o declaratie, unde color este proprietatea si white este valoarea, iar background-color: black; este o alta declaratie.

O nota despre numele fisierelor: styles.css este doar denumirea pe care am ales-o aici. Poti numi fisierul cum doresti, atat timp cat extensia este .css, desi „style” sau „styles” sunt cele mai frecvent utilizate.

Cateva avantaje ale acestei metode sunt:

  1. Pastreaza HTML-ul si CSS-ul separat, ceea ce face ca fisierul HTML sa fie mai mic si sa arate mai curat.
  2. Trebuie sa editam CSS-ul intr-un singur loc, ceea ce este util in special pentru site-urile cu multe pagini care impartasesc stiluri similare.

CSS intern

CSS intern (sau CSS incorporat) presupune adaugarea CSS-ului in interiorul fisierului HTML, in loc sa creezi un fisier complet separat. Cu metoda interna, toate regulile sunt plasate intre o pereche de tag-uri <style> de deschidere si inchidere, care sunt apoi plasate in interiorul tag-urilor <head> din fisierul HTML. Deoarece stilurile sunt introduse direct in tag-urile <head>, nu mai avem nevoie de un element <link> precum in metoda externa.

In afara acestor diferente, sintaxa este exact aceeasi ca in metoda externa (selector, acolade, declaratii): 

                
                    <head>
                    <style>
                        div {
                            color: white;
                            background-color: black;
                        }

                        p {
                            color: red;
                        }
                    </style>
                    </head>
                    <body>
                    ...
                    </body>

Aceasta metoda poate fi utila pentru adaugarea unor stiluri unice unei singure pagini dintr-un site web, dar nu pastreaza separarea dintre HTML si CSS, asa cum face metoda externa, iar in functie de numarul de reguli si declaratii, fisierul HTML poate deveni destul de mare.

CSS inline

CSS inline permite adaugarea stilurilor direct in elementele HTML, desi aceasta metoda nu este atat de recomandata: 

                
                    <body>
                        <div style="color: white; background-color: black;">...</div>
                    </body>

Primul lucru de remarcat este ca nu folosim niciun selector aici, deoarece stilurile sunt adaugate direct in tag-ul de deschidere <div>. Apoi, avem atributul style=, cu valoarea sa in interiorul ghilimelelor, care contine declaratiile.

Daca ai nevoie sa adaugi un stil unic unui singur element, aceasta metoda poate functiona bine. Totusi, in general, nu este recomandata pentru adaugarea CSS la HTML din cateva motive:

  • Poate deveni rapid dezordonata daca adaugi multe declaratii unui singur element, facand fisierul HTML inutil de mare
  • Daca vrei ca mai multe elemente sa aiba acelasi stil, ar trebui sa copiezi si sa lipesti acelasi stil in fiecare element individual, provocand repetitii inutile si mai multa incarcare.
  • Orice CSS inline va suprascrie celelalte doua metode, ceea ce poate duce la rezultate neasteptate. (Desi nu vom explora acest aspect aici, acest lucru poate fi folosit strategic.)
