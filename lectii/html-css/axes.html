Axe in Flexbox

Introducere

Sa vedem cum poate fi controlata orientarea elementelor dintr-un container flex folosind proprietatea flex-direction.

Axe

Cel mai derutant aspect al Flexbox este ca poate functiona atat orizontal, cat si vertical, iar anumite reguli se modifica in functie de directia in care lucrezi. Directia implicita pentru un container flex este orizontala (row), dar o poti schimba pe verticala (column). Poti specifica directia in CSS astfel: 

                
                    .flex-container {
                        flex-direction: column;
                    }

Indiferent de directia pe care o folosesti, trebuie sa te gandesti la containerele flex ca avand 2 axe: axa principala si axa secundara. Directia acestor axe se schimba atunci cand modifici flex-direction.

In majoritatea cazurilor:

  • flex-direction: row → aseaza axa principala orizontal (stanga → dreapta).
  • flex-direction: column → aseaza axa principala vertical (sus → jos).

Cu alte cuvinte, in primul nostru exemplu, am aplicat display: flex pe un <div> si acesta si-a aranjat copiii orizontal. Acesta este un exemplu al comportamentului implicit: flex-direction: row.

Urmatorul exemplu este foarte similar. Daca decomentezi linia flex-direction: column, acele <div>-uri se vor alinia vertical.

Un aspect important de remarcat aici este ca, in acest exemplu, flex-direction: column nu ar functiona corect daca am folosi sintaxa prescurtata flex: 1. Incearca acum (adica modifica valoarea flex pe linia flex: 1 1 auto;). Poti sa-ti dai seama de ce nu functioneaza daca folosim flex: 1? div-urile se prabusesc, desi au clar o inaltime definita acolo.

Motivul este ca sintaxa prescurtata flex extinde flex-basis la 0, ceea ce inseamna ca toate calculele pentru flex-grow si flex-shrink incep de la 0. div-urile goale au implicit o inaltime de 0, asa ca pentru ca elementele flex sa umple inaltimea containerului nu este nevoie sa aiba nicio inaltime predefinita.

Exemplul de mai sus a rezolvat aceasta problema prin specificarea flex: 1 1 auto, permitand elementelor flex sa respecte inaltimea lor data. Am fi putut rezolva problema si:

  • adaugand o inaltime pe containerul .flex-container
  • folosind flex-grow: 1 in loc de sintaxa prescurtata.

Un alt detaliu important: Cand schimbam flex-direction la column, flex-basis se refera la inaltime, nu la latime. In acest context, poate fi evident, dar este ceva de care trebuie sa fim constienti.

Ne-am abatut putin de la subiect… Vorbeam despre flex-direction si axe. Pe scurt:

  • Comportamentul implicit este flex-direction: row, care aranjeaza elementele orizontal.
  • Acest lucru functioneaza bine fara ajustari suplimentare in CSS, deoarece elementele de tip block au implicit latimea containerului parinte.
  • Schimbarea la flex-direction: column adauga complexitate, deoarece elementele de tip block au implicit inaltimea continutului, iar in acest caz nu exista continut.
